# Contributing to DocuShrink AI

First off, thank you for considering contributing to DocuShrink AI! It's people like you that make DocuShrink AI such a great tool.

## 🤝 Code of Conduct
By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## 🚀 Getting Started
1. Fork the repository.
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Data-Processing-System.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Make your changes and commit them: `git commit -m "feat: add some amazing feature"`
6. Push to the branch: `git push origin feature/your-feature-name`
7. Open a Pull Request!

## 🧪 Development Workflow
- **Linting**: Run `npm run lint` to check for style issues.
- **Building**: Run `npm run build` to ensure the production build works.
- **Testing**: Run `npm run test` (if available) to verify logic.

## 📝 Commit Messages
We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `style:` for formatting, missing semi-colons, etc.
- `refactor:` for code changes that neither fix a bug nor add a feature
- `test:` for adding missing tests
- `chore:` for updating build tasks, package manager configs, etc.
